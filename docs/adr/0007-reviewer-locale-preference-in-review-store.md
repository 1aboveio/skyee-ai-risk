# Reviewer Locale Preference In Review Store

The Application Locale is a product-specific reviewer preference for the Skyee AI Risk console, not a general identity-provider profile field. We store the reviewer locale in the Review Store keyed to the signed-in reviewer so server-rendered pages and client interactions can share one preference without coupling this product to identity schema changes. The locale changes product UI only; source evidence values, reviewer notes, raw risk-signal text, transaction descriptions, and saved snapshot contents remain unchanged audit material. When no reviewer preference exists, the app detects browser language first: any Chinese primary language resolves to `zh-CN`, English resolves to `en`, and unsupported or unavailable detection falls back to `zh-CN`. Auth error pages use browser detection/fallback without persisting a reviewer preference, while signed-in surfaces render initially from the stored preference and apply top-header language-menu changes immediately before persisting them in the background.
