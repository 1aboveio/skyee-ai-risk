## 第一层：风险审查（三个组并行，独立运作）

### ① KYC准入与重检
- 工作：身份认证、黑名单扫描、资料审核 → 准入（延续合作）决定
- 执行者：KYC组

### ② 交易规则预警
- 工作：事中规则 + 事后规则，实时监控交易行为
- 执行者：方法架构师/规则管理员

### ③ 实地核查
- 工作：常规核查（初次 + 定期重检）+ 派单核查（风险调查基于风险考量派单）
- 执行者：实地核查组

## 第二层：触发条件（分内部/外部，分深浅度）

### 内部触发

| 触发源 | 子条件 | 深度 |
|--------|--------|------|
| 规则预警 | 事中预警 | 浅度调查 |
| 规则预警 | 事后预警 | 深度调查 |
| 实地查核异常 | 明显造假/重大瑕疵 | 深度调查 |
| 内部举报 | 临时项目排查发现异常（如坏人关联图谱等） | 深度调查 |
| 内部举报 | 公司任何内部举报 | 深度调查 |
| 专项调查 | 管理决策驱动的主动排查（时间窗排查/业务线排查等） | 深度调查 |

### 外部触发

| 触发源 | 子条件 | 深度 |
|--------|--------|------|
| 资金召回 | 不满足以下条件的召回 | 浅度调查 |
| 资金召回 | 3个月内2笔 或 金额异常超1w美金 | 深度调查 |
| 资金召回 | 同一渠道≥2个客户召回（官网不算） | 深度调查（查渠道） |
| RFI异常 | 客户拒绝配合或材料明显瑕疵 | 深度调查 |
| 投诉/账户被盗 | 客户投诉账户被盗 | 深度调查+研发排查 |
| 监管协查 | 被拉黑/监管协查 | 深度调查 |

## 第三层：调查与处置

### 浅度调查
- 查：欺诈嫌疑（合作时长/资金来源/留存率/实地/渠道反馈）、金额大小、非法交易
- 结论：三无→放行归档；有问题→自动升级深度
- 执行者：审查组（初审岗）

### 深度调查
- 在浅度基础上追加：交易行为分析、负面网查、链路还原
- 确认坏人流程：确认坏人 → 关联排查 → 完整报告 → 送复核确认
- 结论：没问题→放行归档
- 执行者：调查组（资深岗）

### 审查原则（四个判断方向）
1. 是否欺诈嫌疑
2. 是否大额洗钱
3. 是否非法交易
4. 渠道要求交易还原证明（独立项，非风险驱动）

### 非法交易界定（SOP用）
**明确非法类：** 枪支/弹药、毒品/精神类药物、赌博/博彩、色情/成人内容
**高风险/需甄别类：** 加密货币/区块链（合规交易所除外；个人钱包/匿名币/混币器行为算）、虚拟资产/NFT（涉嫌洗钱/资金盘/传销模式算）

---

## 第四层：事后处置与沉淀

### 复核确认
- 触发：深度调查确认坏人 → 关联排查 → 完整报告 → 复核
- 复核通过后执行后续处置
- 执行者：复核岗

### 冻结与清退
- 复核确认后，先执行冻结与清退处理（核心动作）
- 执行者：调查组

### 处置出口（冻结清退后执行）
- 系统标签更新（打标记）→ 运营支持
- 反哺规则优化（案件→规则迭代）→ 方法架构师
- 通知合规部门（SAR/STR报送）→ 合规对接岗

### 归档
- 放行案件 → 直接归档，不通知任何人
- 执行者：运营支持

### 坏人原则
- 不通知销售
